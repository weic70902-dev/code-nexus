# JSON 验证工具

JSON 验证工具可以帮助您验证 JSON 数据的有效性，确保数据格式正确无误。

## 功能特性

- JSON 语法验证
- 错误定位和提示
- 行号指示
- 实时验证

## 使用方法

1. 在输入框中粘贴您的 JSON 数据
2. 工具会自动检测数据有效性
3. 如果数据有效，会显示成功消息
4. 如果数据无效，会显示错误信息和位置

## 错误类型

- **语法错误** - 如缺少括号、逗号等
- **格式错误** - 如键名未用双引号包围
- **类型错误** - 如使用了不支持的数据类型

## 示例

### 有效的 JSON
```json
{
  "name": "John",
  "age": 30,
  "city": "New York"
}
```

### 无效的 JSON
```json
{
  "name": "John",
  "age": 30,
  "city": "New York",
  // 缺少闭合大括号
```

### 错误信息示例
```
第 6 行，第 3 列：Unexpected token / in JSON at position 65
```

### 常见错误及修正

#### 1. 使用单引号而不是双引号
```json
// 错误
{'name': 'John', 'age': 30}

// 正确
{"name": "John", "age": 30}
```

#### 2. 缺少逗号分隔符
```json
// 错误
{
  "name": "John"
  "age": 30
}

// 正确
{
  "name": "John",
  "age": 30
}
```

#### 3. 尾随逗号
```json
// 错误
{
  "name": "John",
  "age": 30,
}

// 正确
{
  "name": "John",
  "age": 30
}
```

#### 4. 使用未定义的值
```json
// 错误
{
  "name": undefined,
  "age": 30
}

// 正确（使用 null 代替 undefined）
{
  "name": null,
  "age": 30
}
```

## 应用场景

- **API 开发** - 验证请求和响应数据格式
- **配置文件验证** - 确保 JSON 配置文件语法正确
- **数据导入** - 验证待导入数据的格式
- **调试工具** - 快速定位 JSON 数据中的语法错误

## 注意事项

- 仅支持标准 JSON 格式
- 不支持注释（JSON 标准不支持注释）
